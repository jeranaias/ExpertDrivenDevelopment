Module 1: From User to Builder

Duration: 10 minutes

Bridge from AI Fluency Fundamentals

You have learned to manage AI as a user. Building tools is the same skills at higher complexity. Instead of managing one conversation, you are managing an entire project through AI.

The Builder Mindset

Before you open any tool, decompose on the whiteboard. What are the components? What does the data look like? What does the user need to see? This is Task Decomposition applied to system design.

Instructor Note

Verify all students completed AI Fluency Fundamentals. Ask: “What are the six 201 skills?” If students cannot recall at least four, do a 2-minute refresher.

Module 2: Live Build #1 — Equipment Tracker

Duration: 30 minutes

The Problem

“I need a way to track equipment checkout for my section.”

Step 1: Decompose BEFORE Touching AI (5 min, whiteboard)

  • What data fields? Item, serial number, person, date out, date due, status
  • What does the user need to do? Check out, check in, see overdue
  • Simplest useful version?

Step 2: Build with AI (20 min)

  • Instructor narrates every decision
  • Build working equipment tracker in Power Apps
  • Demonstrate Context Assembly in real time
  • Demonstrate Iterative Refinement: First version functional but ugly, two revision passes

Step 3: Review (5 min)

  • Show finished product
  • “What would you change for your section?”
  • This took 30 minutes. Before AI, this would have been a week-long project.

Instructor Note

Build this live on screen. Narrate every decision as you make it. The point is not a polished product — the point is showing the decompose-then-build workflow in real time.

Module 3: When Something Breaks

Duration: 35 minutes

Purpose

Intentionally create a failure and demonstrate debugging with AI.

Continue building on the equipment tracker. Add a feature slightly outside AI’s comfort zone.

When It Breaks

Let it break. When it does:

  1. Read the error message out loud.
  2. Show the debugging process: paste the error back to AI with context.
  3. Debugging IS the work. This is normal.
  4. Show recovery.

Key Teaching Point

The first time something breaks, most people quit. That is the 80%. Debugging with AI is a skill.

Extended Build

Add 1–2 more features. Full cycle: decompose → build → break → debug → improve.

Instructor Note

Do not fake the failure. Push the build until something genuinely breaks. Authentic debugging is far more instructive than a scripted error. If nothing breaks naturally, add a feature complex enough to force it.

Module 4: Your Problem — Decomposition Exercise

Duration: 25 minutes

Each participant takes their problem and:

  1. Writes a one-sentence problem statement
  2. Decomposes into 4–6 subtasks on paper
  3. Marks each: Human designs / AI builds / Human verifies
  4. Identifies centaur or cyborg pattern for each phase
  5. Identifies potential frontier issues

Peer Review (10 min)

Pair up and review decompositions. “Is any subtask too big? Is there a frontier risk you missed?”

Instructor Note

Walk the room. Quality of decomposition predicts quality of build.

Module 5: Assignment and Path Forward

Duration: 10 minutes

Assignment Before Platform Training

Your Task Before Next Session

  1. Attempt to build your decomposed solution using AI
  2. Document what worked and what did not
  3. Identify at least one failure case to share
  4. Bring a working (or partially working) prototype

Path Forward

Platform Training (4 hr) → Advanced Workshop (4 hr) → Ongoing Contribution

Module 1: The Mindset Shift

Duration: 10 minutes

You are not learning to code. You are learning to describe what you need clearly. The AI handles syntax, structure, and implementation details. Your job is to be the expert on the problem, the process, and the outcome you need.

Good vs. Bad Prompt Examples

Bad Prompt

Prompt — Vague 
Make me an app for leave.

This gives the AI almost nothing to work with. What kind of leave? Who uses it? What does it need to track? What should the output look like?

Good Prompt

Prompt — Specific 
I need a SharePoint list to track leave requests for a 30-person platoon.
Columns: Name (text), Rank (choice: Pvt through GySgt), Start Date (date),
End Date (date), Leave Type (choice: Annual, Emergency, Special Liberty),
Status (choice: Pending, Approved, Denied), Approver (person).
Default the Status to Pending.

This prompt provides context (who uses it), the task (SharePoint list), constraints (specific columns and choices), and format (default values). The AI can build exactly what you need on the first try.

Instructor Note

Spend time on this module. The mindset shift is the single most important concept. If participants leave understanding only one thing, it should be that they are describing problems, not writing code. Use real examples from the unit if possible.

Module 2: How to Talk to AI

Duration: 30 minutes

The Prompt Formula

Every effective prompt has four parts:

The Prompt Formula

Context + Task + Constraints + Format

  • Context: Who are you? What is the situation? What platform are you using?
  • Task: What exactly do you want the AI to build or do?
  • Constraints: What are the limits, rules, or requirements?
  • Format: How should the output be structured or delivered?

Example Prompt Using the Formula

Prompt — Full Formula 
[Context] I'm building a Power App for my platoon's motor pool.
We have 15 vehicles tracked in a SharePoint list called "VehicleInventory"
with columns: BumperNumber, VehicleType, Status, LastPMCS, AssignedDriver.

[Task] Create a gallery screen that shows all vehicles, color-coded by status.

[Constraints] Status values are "FMC" (green), "PMC" (yellow), "NMC" (red).
Only AssignedDriver should be able to edit their vehicle's record.
Must work on mobile phones.

[Format] Give me step-by-step instructions I can follow in Power Apps Studio.

Practice Exercise

Write a prompt for each of these three scenarios. Use the full formula (Context + Task + Constraints + Format):

  1. Scenario 1 — Key Control Log: You need a digital system to replace the paper key control log. It must track who checked out which key, when, and whether it was returned. Only the duty NCO should be able to issue or receive keys.
  2. Scenario 2 — Weekly Training Schedule: Your section needs a shared training schedule that updates automatically. It should show this week's events, the responsible party, location, and required gear. Leadership needs a read-only view.
  3. Scenario 3 — Barracks Room Inspection Tracker: You want to track inspection results by room number, date, inspector, pass/fail, and discrepancies noted. You need to see which rooms have failed more than twice this quarter.

Instructor Note

Give participants 10 minutes to write their prompts, then have 2-3 volunteers share. Walk through each prompt and identify which parts map to Context, Task, Constraints, and Format. Praise specificity. Correct vagueness gently by asking "What would the AI need to know to build this on the first try?"

Module 3: Live Build #1 — Leave Tracker

Duration: 15 minutes

The instructor demonstrates building a leave tracker from scratch using AI, showing the entire prompt-to-working-tool process in real time.

Step 1: Create the Data Structure

Prompt — Step 1 
I need a SharePoint list called "LeaveRequests" for a 30-person platoon.
Columns:
- ServiceMember (text)
- Rank (choice: Pvt, PFC, LCpl, Cpl, Sgt, SSgt, GySgt)
- StartDate (date)
- EndDate (date)
- LeaveType (choice: Annual, Emergency, Special Liberty, Convalescent)
- Status (choice: Pending, Approved, Denied) - default to "Pending"
- ApprovedBy (person)
- Notes (multiline text)

Give me the exact steps to create this in SharePoint.

Step 2: Build the Input Form

Prompt — Step 2 
Now I need a Power Apps canvas app connected to the "LeaveRequests"
SharePoint list. Create a form screen where Marines can submit leave requests.
The form should auto-fill the current user's name. StartDate must be at
least 2 days in the future. Show a confirmation message after submission.

Step 3: Build the Status View

Prompt — Step 3 
Add a second screen to the leave tracker app that shows a gallery of all
leave requests. Color-code by status: Pending (yellow), Approved (green),
Denied (red). Add a filter dropdown to show only a specific status.
Sort by StartDate descending.

Instructor Note

Build this live on screen. Copy-paste the prompts into your AI tool and show the raw output. Do not pre-build anything. The point is for participants to see that AI output is imperfect but fast. If the AI makes a mistake, even better — that sets up Module 4 on iteration.

Module 4: The Iteration Loop

Duration: 30 minutes

AI will not give you a perfect answer every time. That is expected. The skill is knowing how to respond when the output is wrong. Here are four strategies:

Strategy 1: Be More Specific

If the output is close but not right, add more detail to your prompt.

Prompt — Add Specificity 
The gallery is showing all leave requests, but I only want to show requests
where the Status is "Pending" and the ServiceMember matches the current
logged-in user. Filter the gallery's Items property to show only those records.

Strategy 2: Show the Error

Copy the exact error message and paste it into your next prompt.

Prompt — Share Error 
I'm getting this error in Power Apps:
"The function 'Filter' has some invalid arguments."
It happens on the gallery's Items property which is currently set to:
Filter(LeaveRequests, Status = "Pending")
What is wrong and how do I fix it?

Strategy 3: Break It Down

If you asked for too much at once, split your request into smaller pieces.

Prompt — Smaller Steps 
Let's just focus on the filter. How do I write a Filter formula in
Power Apps that filters a SharePoint list called "LeaveRequests" to
show only records where the Status column equals "Pending"?

Strategy 4: Start Fresh

If the conversation has gone off track, start a new chat. Summarize what you have so far and what you need next.

Prompt — Fresh Start 
I'm starting over. I have a Power Apps canvas app connected to a
SharePoint list called "LeaveRequests" with these columns: [list columns].
The app currently has a form screen that works. I need help building a
gallery screen that shows filtered results. Let's start with just
displaying all records in a simple gallery.

Practice Exercise

The instructor will show a broken prompt and its incorrect AI output. Working in pairs, participants choose one of the four strategies and write a follow-up prompt to fix the issue. Share with the group and discuss which strategy works best for each situation.

Instructor Note

Prepare 2-3 examples of broken AI output before class. Intentionally bad output is more memorable. Emphasize that iteration is normal and expected — professional developers iterate constantly. The goal is not a perfect first prompt; the goal is a working tool within 30 minutes.

Module 5: Live Build #2 — Leave Tracker (Continued)

Duration: 20 minutes

Continue the leave tracker from Module 3, adding filtering and automation to demonstrate a complete, usable tool.

Add Filtering

Prompt — Filtering 
Add a dropdown at the top of the gallery screen that lets the user filter
by LeaveType (Annual, Emergency, Special Liberty, Convalescent) and a
second dropdown for Status (Pending, Approved, Denied). Include an
"All" option in each dropdown. The gallery should update dynamically
as the user selects filters.

Add Email Notification

Prompt — Automation 
Create a Power Automate flow that triggers when a new item is added to
the "LeaveRequests" SharePoint list. The flow should send an email to
the platoon sergeant (use a specific email address as a variable) with
the subject "New Leave Request: [ServiceMember Name]" and a body that
includes the member's name, rank, leave type, start date, and end date.

Instructor Note

This module shows that building useful tools is incremental. Each prompt adds one piece of functionality. Remind participants that they do not need to build everything in one session. Ship something small, get feedback, iterate.

Wrap-Up & Assignment

Duration: 5 minutes

Key Takeaways

  • You are describing problems, not writing code.
  • Use the Prompt Formula: Context + Task + Constraints + Format.
  • Iteration is normal. Use the four strategies when AI gives wrong output.
  • Build incrementally. Ship small, get feedback, improve.

Assignment

Your Task Before Next Session

Pick ONE real problem in your section. Write a prompt using the full formula (Context + Task + Constraints + Format). Try building it with an AI tool. Bring your prompt and result (working or not) to the next training session.

Platform Orientation: Where Things Live

Duration: 15 minutes

Before building, everyone needs to know where the tools are and how they connect. Here is the platform landscape:

Platform Purpose Access URL
SharePoint Online Data storage — lists and document libraries https://[tenant].sharepoint.com
Power Apps App builder — forms, galleries, screens https://make.powerapps.com
Power Automate Workflow engine — triggers, approvals, emails https://make.powerautomate.com
Power BI Dashboards — charts, reports, data visualization https://app.powerbi.com

Instructor Note

Have everyone log in to each platform before proceeding. Verify access now so you do not lose time during the builds. If anyone has access issues, have IT support contact information ready. Allow 5 extra minutes if needed.

Build #1: Equipment Tracker

Duration: 60 minutes  |  Platforms: SharePoint + Power Apps

Phase 1: Data — Create the SharePoint List

Prompt — Data Layer 
I need a SharePoint list called "EquipmentTracker" for a company-level
motor pool with 40 vehicles and 200 pieces of serialized gear.

Columns:
- ItemName (text) - e.g., "HMMWV" or "PVS-14"
- SerialNumber (text) - unique
- Category (choice: Vehicle, Optics, Communications, Weapons, Other)
- Status (choice: FMC, PMC, NMC, In Maintenance) - default "FMC"
- AssignedTo (person)
- Location (choice: Motor Pool, Armory, Field, In Transit)
- LastInventoryDate (date)
- Notes (multiline text)

Give me exact steps to create this in SharePoint Online.

Phase 2: App — Build the Power App

Prompt — App Build 
Create a Power Apps canvas app connected to the "EquipmentTracker"
SharePoint list. I need:

Screen 1 - Gallery: Show all equipment in a scrollable list. Display
ItemName, SerialNumber, Category, and Status. Color-code the Status
field: FMC (green), PMC (yellow), NMC (red), In Maintenance (blue).
Add a search bar that filters by ItemName or SerialNumber.

Screen 2 - Detail/Edit: When a user taps an item in the gallery,
navigate to a detail screen that shows all fields and allows editing.
Only the AssignedTo person or an admin should be able to edit.

Give me step-by-step instructions for Power Apps Studio.

Phase 3: Polish — Add Usability Features

Prompt — Polish 
Add these improvements to the Equipment Tracker app:

1. A summary bar at the top of the gallery screen showing counts:
   "FMC: [count] | PMC: [count] | NMC: [count] | In Maintenance: [count]"
2. A filter dropdown for Category so users can view only Vehicles or
   only Optics, etc. Include an "All" option.
3. A "Last Inventory" warning: if LastInventoryDate is more than 30 days
   ago, show a red banner on that item's card saying "INVENTORY OVERDUE".

Give me the Power Apps formulas for each feature.

Instructor Note

Walk the room during this build. Most common issues: (1) SharePoint column type mismatches, (2) forgetting to connect the data source in Power Apps, (3) formula syntax errors in gallery Items property. Have participants help each other before you intervene. Peer troubleshooting builds confidence.

Build #2: Request Workflow

Duration: 60 minutes  |  Platform: Power Automate

Phase 1: Trigger — Set Up the Flow

Prompt — Flow Trigger 
Create a Power Automate flow that triggers when a new item is created
in a SharePoint list called "SupplyRequests". The list has these columns:
- RequestedBy (person)
- ItemDescription (text)
- Quantity (number)
- Priority (choice: Routine, Urgent, Emergency)
- Justification (multiline text)
- Status (choice: Pending, Approved, Denied) - defaults to "Pending"
- ApprovedBy (person)

The flow should:
1. Trigger on new item creation
2. Send an email notification to the supply chief with request details
3. Include the requester's name, item description, quantity, priority,
   and justification in the email body

Give me step-by-step Power Automate instructions.

Phase 2: Approval — Add Approval Logic

Prompt — Approval Chain 
Extend the SupplyRequests flow to include an approval step:

1. After the notification email, add an Approval action (Approve/Reject)
   assigned to the supply chief.
2. If approved: update the SharePoint item Status to "Approved", set
   ApprovedBy to the approver, and send a confirmation email to the requester.
3. If rejected: update Status to "Denied" and send a rejection email to
   the requester with the approver's comments.

Show me how to configure the approval and the conditional branches.

Phase 3: Error Handling — Make It Robust

Prompt — Error Handling 
Add error handling to the SupplyRequests approval flow:

1. Add a "Configure run after" on the approval step so that if it fails
   or times out, the flow sends an alert email to the admin.
2. Set a 48-hour timeout on the approval. If no response in 48 hours,
   automatically escalate by sending the approval to the next-level
   supervisor.
3. Add a "Scope" action around the entire flow with a try/catch pattern:
   if any action fails, log the error details to a SharePoint list
   called "FlowErrorLog".

Give me the Power Automate configuration steps.

Instructor Note

Power Automate is where most participants will struggle. The visual flow designer is different from Power Apps. Common issues: (1) wrong dynamic content selected, (2) approval connector not configured, (3) "Configure run after" settings missed. Demo the first phase live, then let participants try phases 2-3 with their AI tool.

Build #3: Leadership Dashboard

Duration: 60 minutes  |  Platform: Power BI

Phase 1: Connect — Data Sources

Prompt — Data Connection 
I need to create a Power BI dashboard that pulls data from three
SharePoint lists:
- "EquipmentTracker" (ItemName, SerialNumber, Category, Status, Location)
- "LeaveRequests" (ServiceMember, Rank, StartDate, EndDate, Status)
- "SupplyRequests" (RequestedBy, ItemDescription, Priority, Status)

Walk me through connecting Power BI Desktop to these three SharePoint
Online lists. Show me how to set up the data connections and verify
the data loads correctly.

Phase 2: Visuals — Build the Charts

Prompt — Dashboard Visuals 
Create a Power BI dashboard page called "Commander's Overview" with
these visuals:

1. Equipment Readiness: A donut chart showing equipment by Status
   (FMC, PMC, NMC, In Maintenance). Use green, yellow, red, and blue.
2. Leave Calendar: A bar chart showing the number of personnel on leave
   per week for the next 8 weeks.
3. Supply Pipeline: A stacked bar chart showing supply requests by
   Priority (Routine, Urgent, Emergency) and Status (Pending, Approved,
   Denied).
4. Key Metrics Cards: Four card visuals showing:
   - Total Equipment FMC Rate (percentage)
   - Personnel Currently on Leave (count)
   - Open Supply Requests (count)
   - Overdue Equipment Inventories (count where LastInventoryDate > 30 days ago)

Give me step-by-step Power BI instructions for each visual.

Phase 3: Useful — Filters and Sharing

Prompt — Usability 
Make the Commander's Overview dashboard more useful:

1. Add a date range slicer that filters the Leave and Supply data
   by a selectable time period.
2. Add a Category slicer for the Equipment data so leadership can
   view only Vehicles or only Communications gear.
3. Set up automatic data refresh so the dashboard updates every
   morning at 0600.
4. Publish the dashboard to the Power BI service and share it with
   a security group called "CompanyLeadership" as read-only viewers.

Show me the steps for each improvement.

Instructor Note

Power BI is the most visual and rewarding build. Participants will see immediate results as charts populate. Common issues: (1) data type mismatches from SharePoint, (2) DAX formula errors in calculated columns, (3) publishing permissions. If time is short, focus on Phase 1 and Phase 2 — publishing can be done after class.

Troubleshooting Patterns

Duration: 15 minutes

When something breaks, these three prompt patterns will solve most issues. Save them as templates.

Pattern 1: The Error Reporter

Prompt — Error Report 
I'm working in [Power Apps / Power Automate / Power BI].
I'm trying to [describe what you were doing].
I got this error: "[paste exact error message]"
Here is the formula/configuration I'm using: [paste it]
What is wrong and how do I fix it?

Pattern 2: The Behavior Describer

Prompt — Wrong Behavior 
My [app/flow/dashboard] is doing [describe what it is doing].
I expected it to do [describe what you wanted].
Here is my current configuration: [paste relevant settings].
What do I need to change?

Pattern 3: The Fresh Eyes Request

Prompt — Review Request 
I've been going back and forth trying to fix this and I'm stuck.
Here is what I'm building: [brief description].
Here is what works so far: [list working parts].
Here is what is broken: [describe the problem].
Can you look at this with fresh eyes and suggest a different approach?

Instructor Note

These three patterns cover 90% of troubleshooting situations. Have participants save these as notes or bookmarks. Reinforce that the key skill is describing what happened clearly — the same mindset from the Orientation course.
